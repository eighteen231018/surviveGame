**# ROBLOX生存挑战 - 游戏开发作业需求文档**





\## 一、游戏概述


\#技术要求

工程与同步

◦ 使用 Rojo 实现本地代码与 Roblox Studio 双向同步。

◦ 项目结构清晰，脚本按客户端/服务端规范存放，不随意乱放脚本。

◦ 项目代码提交至 Git 仓库，包含合理的提交记录，不包含冗余/无用文件。

项目中包含roblox place 文件，用于在roblox studio中打开游戏场景。


\### 1.1 游戏名称



生存挑战（Survival Challenge）



\### 1.2 游戏类型



开发世界生存冒险/塔防/肉鸽皆可



\### 1.3 游戏目标



玩家在一个充满怪物的开放世界中生存，采集资源、制作装备、抵御怪物巢穴的进攻波次，最终摧毁所有怪物巢穴获得胜利。



\### 1.4 核心体验



\- 探索开放世界，采集资源

\- 与智能怪物战斗，体验流畅的战斗手感

\- 使用多样化的道具丰富游戏策略

\- 抵御越来越难的怪物波次进攻





\## 二、玩家系统





\### 2.1 玩家属性



玩家拥有一系列属性，所有属性的\*\*初始值从配置文件读取\*\*，游戏过程中可通过道具、升级等方式改变,属性设计参考：



| 属性 | 说明 | 影响效果 |

|------|------|----------|

| \*\*生命值 (Health)\*\* | 玩家的生命值，降为0则死亡 | 无 |

| \*\*力量 (Strength)\*\* | 影响攻击力 | 力量越高，攻击力越高 |

| \*\*敏捷 (Agility)\*\* | 影响攻击间隔和移动速度 | 敏捷越高，攻击间隔越短、移动速度越快 |

| \*\*智力 (Intelligence)\*\* | 影响物品使用效果 | 智力越高，药水回复效果越好、炸弹伤害越高 |



\#### 要求：



\- 属性系统必须支持\*\*配置文件驱动\*\*（修改配置文件即可调整初始属性和成长曲线）

\- 属性变化需要\*\*实时生效\*\*（如增加力量后，下次攻击立即生效;被怪物施加减速负面效果立即减少移动速度）

\- 属性要存档



\### 2.3 玩家死亡与复活



\- 玩家死亡后，在基地复活

\- 死亡时\*\*掉落当前持有的物品\*\*（可选：掉落背包中的随机物品）

\- 复活后属性恢复到死亡前的值

\- 死亡次数影响最终结算得分





\## 三、怪物巢穴系统



怪物巢穴是怪物的生成点，也是玩家的攻击目标。需要可在地图中\*\*自定义摆放\*\*多个怪物巢穴。



\#### 要求：



\- 巢穴必须支持\*\*模块化配置\*\*（通过配置文件定义巢穴属性,如怪物品种\\生成周期等）

\- 可以在地图编辑模式下，将巢穴放置在任意位置

\- 巢穴生成后，在游戏中显示为可见的建筑物/洞穴/孵化器等



\- 配置文件使用\*\*结构化格式\*\*，方便策划调整数值

\- 支持\*\*多种怪物混合生成\*\*（如巢穴可以同时生成蝎子和蜘蛛，按权重随机）

\- 支持\*\*生成规则配置\*\*（如随机生成/顺序生成/权重随机生成）







\## 四、怪物系统





\### 4.1 怪物类型



设计至少3种怪物类型，每种怪物有不同的属性和行为特点：



| 怪物类型 | 特点 | 攻击方式 | 移动速度 |

|----------|------|----------|----------|

| \*\*蝎子\*\* | 高攻击、低血量 | 近战尾刺攻击 | 中等 | 会钻地迂回\\可能施加毒素攻击

| \*\*蜘蛛\*\* | 低攻击、高血量 | 近战撕咬攻击 | 快 | 会群体进攻\\可能把玩家网住不能移动几秒

| \*\*Boss怪物\*\* | 超高攻击、超高血量 | 近战+范围攻击 | 慢  |行为模式多样化



\### 4.2 怪物属性



每种怪物支持以下可配置属性参考：



| 属性 | 说明 | 示例值 |

|------|------|--------|

| \*\*生命值\*\* | 怪物生命值 | 100 |

| \*\*攻击力\*\* | 每次攻击造成的伤害 | 15 |

| \*\*攻击范围\*\* | 攻击距离 | 5 |

| \*\*攻击间隔\*\* | 两次攻击之间的冷却时间（秒） | 2 |

| \*\*攻击前摇\*\* | 攻击动作开始到伤害判定的时间（秒） | 0.3 |

| \*\*攻击后摇\*\* | 攻击动作结束后到可以行动的间隔时间（秒） | 0.5 |

| \*\*移动速度\*\* | 行走速度 | 16 |

| \*\*转身速度\*\* | 转身速度（度/秒） | 180 |

| \*\*警戒范围\*\* | 发现玩家的距离 | 25 |

| \*\*追击范围\*\* | 追击玩家的最大距离，超出后放弃追击 | 50 |

| \*\*是否主动攻击\*\* | 是否主动攻击玩家 | `true` / `false` |

| \*\*经验值\*\* | 击杀后玩家获得的经验 | 20 |



\#### 要求：



\- 怪物属性必须从配置文件读取，支持快速调整数值

\- 不同怪物类型使用不同的配置文件



\### 4.3 怪物AI行为



怪物需要拥有\*\*一定的智能\*\*的行为模式，确保战斗体验顺畅自然。



\#### 要求：



\- AI行为需要\*\*流畅自然\*\*，不能有突兀的转身或卡住现象

\- 怪物不能\*\*穿墙或卡在障碍物中\*\*

\- 多只怪物同时攻击时，不能全部挤在玩家面前，需要有\*\*包围和迂回行为\*\*



\### 4.4 怪物掉落系统



怪物死亡后，有概率掉落道具。



\#### 要求：



\- 掉落系统必须支持\*\*配置文件驱动\*\*

\- 掉落的物品需要在地面生成，玩家可以走近自动拾取





\## 五、物品系统





\### 5.1 物品概述



游戏世界中将放置一些物品，怪物死亡后也会掉落物品。玩家可以拾取、使用、丢弃物品。



\#### 要求：



\- 物品配置必须支持\*\*参数驱动\*\*（通过配置文件定义物品属性）

\- 物品系统最好支持\*\*组件化设计\*\*（通过配置不同的组件就能实现不同的物品特性,而不是为每个物品硬编码实现逻辑）

\- 可选:支持\*\*物品堆叠\*\*（同类型物品可以堆叠在一起，减少背包占用）



\### 5.2 物品类型设计参考



设计以下6种物品类型，每种物品有独特的玩法功能：



\#### 1. 长剑 (Long Sword)



\- \*\*类型\*\*：武器

\- \*\*功能\*\*：近战攻击武器

\- \*\*配置参数\*\*：

&#x20; - 攻击力：25

&#x20; - 攻击范围：6

&#x20; - 攻击间隔：1.2秒

&#x20; - 耐久度：150（每次攻击消耗1点）

\- \*\*使用效果\*\*：玩家装备后，左键点击攻击怪物

\- \*\*特殊效果\*\*：无



\#### 2. 大锤 (Great Hammer)



\- \*\*类型\*\*：武器

\- \*\*功能\*\*：重型近战攻击武器

\- \*\*配置参数\*\*：

&#x20; - 攻击力：50

&#x20; - 攻击范围：8

&#x20; - 攻击间隔：2.5秒

&#x20; - 耐久度：100（每次攻击消耗2点）

\- \*\*使用效果\*\*：玩家装备后，左键点击攻击怪物

\- \*\*特殊效果\*\*：攻击有概率造成眩晕效果（怪物短时间内无法行动）



\#### 3. 炸弹 (Bomb)



\- \*\*类型\*\*：消耗品

\- \*\*功能\*\*：投掷后爆炸，对范围内所有怪物造成伤害

\- \*\*配置参数\*\*：

&#x20; - 爆炸伤害：40

&#x20; - 爆炸范围：10

&#x20; - 引爆延迟：3秒

&#x20; - 投掷距离：15

\- \*\*使用效果\*\*：玩家右键点击使用，投掷炸弹到目标位置，3秒后爆炸

\- \*\*特殊效果\*\*：爆炸有概率击退怪物



\#### 4. 生命药水 (Health Potion)



\- \*\*类型\*\*：消耗品

\- \*\*功能\*\*：恢复玩家生命值

\- \*\*配置参数\*\*：

&#x20; - 恢复量：30（基础值，受智力属性影响）

&#x20; - 使用时间：1秒

&#x20; - 堆叠上限：10

\- \*\*使用效果\*\*：玩家右键点击使用，播放喝药动画，恢复生命值

\- \*\*特殊效果\*\*：智力属性越高，恢复量越多



\#### 5. 隐形药水 (Invisibility Potion)



\- \*\*类型\*\*：消耗品

\- \*\*功能\*\*：使玩家隐身一段时间，怪物无法发现玩家

\- \*\*配置参数\*\*：

&#x20; - 隐身持续时间：10秒

&#x20; - 使用时间：1秒

&#x20; - 堆叠上限：5

\- \*\*使用效果\*\*：玩家右键点击使用，播放特效，怪物失去追击目标

\- \*\*特殊效果\*\*：受到攻击或攻击他人时，隐身效果消失



\#### 6. 箱子 (Chest)



\- \*\*类型\*\*：容器

\- \*\*功能\*\*：可以存储物品的容器,保护物品不被怪物破坏，放置在世界中

\- \*\*配置参数\*\*：

&#x20; - 格子数量：10

&#x20; - 可交互距离：8

\- \*\*使用效果\*\*：玩家靠近箱子，按E键打开UI，存放/取出物品

\- \*\*特殊效果\*\*：箱子不会消失，重启游戏后物品仍在



\### 5.3 物品配置参数参考



每种物品支持以下通用配置参数：



| 参数 | 说明 | 示例值 |

|------|------|--------|

| \*\*物品ID\*\* | 唯一标识符 | `Sword` |

| \*\*物品名称\*\* | 显示名称 | "长剑" |

| \*\*物品类型\*\* | 武器/消耗品/容器等 | `Weapon` |

| \*\*模型ID\*\* | 3D模型资源ID | `SwordModel` |

| \*\*图标ID\*\* | UI图标资源ID | `SwordIcon` |

| \*\*标签\*\* | 物品标签（用于分类和识别） | `Weapon`, `Melee` |

| \*\*重量\*\* | 物品重量，影响玩家移动速度 | 2 |

| \*\*最大堆叠数\*\* | 同类型物品最多堆叠多少个 | 10 |

| \*\*是否可丢弃\*\* | 是否可以丢弃到地上 | `true` |

| \*\*是否可拾取\*\* | 是否可以通过触碰自动拾取 | `true` |

| \*\*是否可装备\*\* | 是否可以装备到手上 | `true` |

| \*\*是否可自动装备\*\* | 是否可以一捡起来就装备到手上否则进背包 | `true` |

| \*\*组件配置\*\* | 物品的功能组件配置 | `Attacker={...}` |







\## 六、战斗系统





\### 6.1 伤害计算



伤害计算公式参考（学生需要设计合理的公式）：



最终伤害 = 基础攻击力 × (1 + 力量/100) × (1 + 伤害类型加成/1000) - 目标防御力





\### 6.2 战斗体验

\- \*\*攻击判定\*\*：准确,手动舒服

\- \*\*攻击动画\*\*：自然流畅

\- \*\*受击反馈\*\*：被攻击时，目标需要有明显的受击反应（动画、音效、特效）





\## 七、数据持久化系统





\### 7.1 需要保存的数据



游戏进度需要保存，玩家下次进入游戏时，恢复上次的进度。需要保存的数据包括：



\#### 玩家数据



| 数据 | 说明 |

|------|------|

| \*\*位置\*\* | 玩家当前位置 |

| \*\*属性\*\* | 生命值、力量、敏捷、智力等 |

| \*\*背包物品\*\* | 玩家背包中的所有物品（ID、堆叠数、耐久度等） |

| \*\*装备物品\*\* | 当前装备的武器/道具 |



\#### 世界数据



| 数据 | 说明 |

|------|------|

| \*\*世界物品\*\* | 放置在世界中的物品（如箱子、采集物等） |

| \*\*怪物巢穴\*\* | 已生成的怪物巢穴位置和状态 |

| \*\*怪物波次\*\* | 当前波次、已摧毁的巢穴数量等 |



\### 7.2 数据结构设计



学生需要自行设计合理的数据结构，用于存储和加载游戏进度。



\#### 要求：



\- 数据需要\*\*分模块管理\*\*（玩家数据、世界数据分开存储）

\- 使用 Roblox官方提供的数据存储服务 

\- 数据存储需要\*\*稳定可靠\*\*，不能丢失玩家进度

\- 需要支持\*\*多玩家同时游戏\*\*（如果是在线多人游戏）

\## 九、技术需求





\### 9.1 配置驱动开发



\- 所有游戏参数（玩家属性、怪物属性、物品属性等）必须通过\*\*配置文件\*\*定义

\- 修改配置文件后，游戏内数值需要\*\*实时生效\*\*（热更新）或\*\*重启后生效\*\*

\- 配置文件需要\*\*结构清晰\*\*、\*\*易于修改\*\*



\### 9.2 模块化设计



\- 怪物巢穴需要支持\*\*模块化配置\*\*，方便扩展新的巢穴类型

\- 物品系统最好支持\*\*组件化设计\*\*，方便扩展新的物品功能

\- 代码需要\*\*高内聚、低耦合\*\*，避免硬编码



\### 9.3 性能优化



\- 怪物AI不能\*\*占用过多CPU\*\*，需要优化寻路和状态判断

\- 物品堆叠需要\*\*减少实体数量\*\*，同类型物品尽量合并

\- 数据保存不能\*\*阻塞主线程\*\*，需要使用异步保存

|--------|------|------|

| \*\*功能完整性\*\* | 40分 | 所有需求功能是否实现、是否可正常运行 |

\- 功能有Bug导致游戏崩溃（-10分）

\- 配置参数硬编码在代码中，无法通过配置文件修改（-5分）

\- 未提交设计文档或测试报告（-10分）
